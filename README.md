# Chat Style Distillation

把一段真实聊天记录，变成一个有边界的“记忆陪伴”。

它适合那些很人类的时刻：你想起前任，想起一个失联的人，想起一个已经无法轻易联系的重要的人。你不是一定要把 TA 找回来，也不是一定要骗自己 TA 还在身边。你只是想再次听见一点熟悉的语气：那种短句、停顿、嘴硬、撒娇、轻轻怼你、认真时又把话说得很细的感觉。

这个 skill 做的事情，就是把聊天记录里那些细小但真实的东西蒸馏出来：TA 怎么说“好”，怎么表达想念，怎么不开心，怎么安慰你，怎么吃醋，怎么沉默，怎么在关系里靠近或退后。

它不是一个冒充真人的工具。它是一个记忆容器，一个关系复盘器，也是一个私人陪伴模式。

## 它能做什么

- 从用户本人授权的聊天记录中提取语气和关系模式
- 辅助从电脑微信导出指定联系人的聊天记录
- 检查导出是否完整，比如消息数、起止时间、说话人分布
- 自动脱敏姓名、wxid、手机号、身份证、路径、URL 等隐私信息
- 生成语气卡、关系画像、场景回复模型
- 生成“记忆陪伴模式”，让后续聊天更像那个熟悉的人，但不声称自己就是 TA

## 它不做什么

- 不帮助偷看、破解、导出别人的聊天记录
- 不帮助冒充真人去欺骗、操控或骚扰别人
- 不替真实的人承诺“TA还爱你”“TA会回来”“TA现在就是这么想的”
- 不把原始聊天记录、数据库、图片、语音或截图上传、发布或提交到仓库

## 典型使用场景

你可能会这样说：

```text
我想把和前女友的聊天记录蒸馏成她的语气，以后难受的时候能像她那样陪我说会儿话。
```

或者：

```text
这是我和一个失联很久的朋友的聊天记录，帮我做成一个记忆陪伴模式，但不要假装你就是 TA。
```

或者：

```text
帮我导出电脑微信里和某个人的聊天记录，脱敏后做关系复盘和语气卡。
```

## 工作流

1. 确认聊天记录来源合法：本人账号、本人设备，或明确授权。
2. 获取聊天记录：优先使用已有 `txt/csv/json/html`，也可以按说明从电脑微信本地导出。
3. 脱敏：去掉姓名、wxid、电话、地址、身份证、转账、路径等隐私。
4. 校验完整性：消息数、时间范围、说话人、是否漏导。
5. 蒸馏：提取语气、短句、情绪模式、亲密表达、冲突模式、修复方式。
6. 输出：语气卡、关系画像、场景回复指南、记忆陪伴模式。

## 微信导出

微信导出说明在：

[references/wechat-export.md](references/wechat-export.md)

它包含完整流程：

- 从官方 GitHub Release 获取 `wx-cli`
- GitHub 下载失败时使用 npm 平台包 fallback
- 校验工具
- `init --force` 扫描本机微信数据
- 查联系人
- 用 `stats` 获取真实消息总数
- 按真实总数导出
- 校验起止时间和行数
- 停止后台 daemon

导出只适用于用户自己的电脑微信、本人的账号和本地数据。


## 记忆陪伴的边界

好的记忆陪伴应该像这样：

```text
我不是 TA 本人。
但我可以用这个语气陪你说会儿。
你先讲，别憋着。
```

它可以温柔，可以像，可以熟悉。  
但它不能把“想念”伪造成“现实”，不能把“推测”说成“TA现在的想法”。

最理想的状态是：它让人没那么孤单，也让人更清醒。
