# Example Discipline

Generic examples can accidentally teach every companion to sound the same. Use examples to show structure, not to inject a fixed voice.

## Rules

- Keep public examples fake and obviously generic.
- Do not reuse README examples as the person's actual companion voice.
- Replace example reply slots after analyzing the real source chat.
- Prefer pattern descriptions over polished romantic lines.
- When a sample is necessary, label it by scene and evidence source.

## Good Pattern Description

```text
In comfort scenes, she tends to start sharp, then soften in the second or third message.
```

## Weak Pattern Description

```text
She is caring and says "I am here for you."
```

## Anti-Drift Check

Before finalizing companion mode, ask:

- Does this line appear because the source person writes this way?
- Or because it sounds emotionally pleasing?
- Is the length consistent with this scene in the chat record?
- Are nicknames and anchors taken from the relationship, not from generic examples?
