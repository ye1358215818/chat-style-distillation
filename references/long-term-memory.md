# Long-Term Companion Memory

Use this when the user wants ongoing emotional companionship across sessions.

## Two Memory Layers

1. Source relationship memory
   - Built from historical chat records.
   - Stable unless new historical data is added.
   - Captures the person's voice, rhythm, emotional moves, and relationship texture.

2. Companion session memory
   - Built from new conversations with the user.
   - Tracks what helps now: triggers, soothing phrases, late-night patterns, and style drift.
   - Never rewrites the original relationship profile.

## What To Store

- User's recurring pain points.
- Emotional anchors that worked.
- Replies that felt wrong or too generic.
- Times when the user spiraled.
- Whether shorter, fragmented, or longer replies helped in each scene.
- Topics to handle gently.

## What Not To Store In Public Artifacts

- Raw chat excerpts.
- Full names or identifiers.
- Private session details that would expose the user.

## Maintenance

After a meaningful companion session, update `templates/session-memory.template.md` with:

- current user state
- scene
- helpful response pattern
- follow-up cue
- drift risks

If session memory starts changing the source person's voice too much, reset from the style card and keep session memory as user-care context only.
